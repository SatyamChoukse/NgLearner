Related Concepts

  • S

    Signal

    A container for a value. When the value changes, it tells Angular to update everything that depends on it.

  • C

    Computed

    Computed signals are read-only signals that derive their value from other signals. They update automatically when dependencies change.

  • E

    Effect

    Code that runs automatically when a signal changes (e.g., logging to the console or calling an API).

Why Use Signals?

  • Easy Updates - Signals automatically keep your app in sync.

  • No Manual Watching - No need for subscribe or unsubscribe like RxJS.

  • Clean Code - Everything is simple and readable.

Traditional vs. Signals

Traditional Angular With Signals
Uses Zone.js to detect changes in the entire app. Signals directly notify only the affected parts.
All templates are checked for updates. Only the template using the signal is updated.
Manual subscriptions (e.g., with RxJS) are needed. No subscriptionsâ€”signals manage everything.
